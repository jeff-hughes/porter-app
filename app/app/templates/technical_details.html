Technical Details

The online version of this family history was first generated by scanning in the typewritten version, and using optical character recognition (OCR) – via pyocr and tesseract – to convert the typed pages into plain text. (The pages were first converted from PDF to jpeg images using the Python Image Library, or Pillow.) This resulted in mostly usable text, but still required fairly significant manual cleaning to correct errors.

Creating a script to properly parse all of the details in the text proved challenging, and ultimately the text was manually inputted into three CSV files, designating people, children, and marriages. From there, it was ready to be inserted into the database.

The data is stored in a neo4j database, a graph database that allows for sophisticated ways to navigate relationships between data points. Given that a family tree constitutes a graph of nodes and edges, neo4j seemed like a natural fit. (Note that, for this relatively small amount of data, the choice of database is inconsequential; I also just wanted a chance to play around with a graph database.) The database is then accessed via the py2neo package, which provides a Python toolkit for accessing neo4j databases.

The app itself uses Flask, a Python web framework, and both the Flask app and the database are set up in Docker containers for easier maintenance and dependency management.

Code for the app is available on Github.
